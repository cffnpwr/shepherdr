# サービス管理

## 設定ファイル

サービス定義は`~/.config/shepherdr/config.toml`に置く。
記述例は[config.example.toml](./config.example.toml)を参照。

`[[services]]`の各エントリは以下のフィールドを持つ。

| フィールド | 必須 | 型 | 内容 |
| --- | --- | --- | --- |
| `name` | ○ | 文字列 | サービス名。全エントリで一意。ログファイル名・UI表示に使う |
| `command` | ○ | 文字列配列 | 実行するargv。シェル評価しない |
| `login_shell` | - | 真偽値 | `true`でログインシェル経由で起動する。既定は`false` |
| `env` | - | テーブル | 追加の環境変数。appプロセスの環境に上乗せする |
| `cwd` | - | 文字列 | 作業ディレクトリ。未指定ならappプロセスのものを継承する |
| `enabled` | - | 真偽値 | `false`で定義を残したまま自動起動を止める。既定は`true` |

任意でトップレベルに`[log]`セクションを置き、ログローテーションの上限を上書きできる。セクション自体の省略、および各フィールドの個別省略も可能で、省略した項目は実装の既定値を使う。

| フィールド | 必須 | 型 | 内容 |
| --- | --- | --- | --- |
| `max_size` | - | 文字列 | 1世代あたりのファイルサイズ上限。`"10MiB"`のように単位付きで指定する。2進系（`KiB`/`MiB`/`GiB`＝1024単位）・10進系（`KB`/`MB`/`GB`＝1000単位）のどちらも受け付け、単位は大文字小文字を区別しない。正の値のみ有効。省略時は実装の既定値 |
| `max_generations` | - | 整数 | 保持する世代数（現在のファイルを含む）。1以上の値のみ有効。省略時は実装の既定値 |

トップレベルの`[restart]`セクション（任意）で、再起動ポリシー（後述「監視と再起動」参照）の既定値を上書きできる。省略したフィールド・セクション自体の省略は実装の既定値に従う。時間を表すフィールドは`"1s"`・`"500ms"`・`"1h 30m"`のように数値＋単位の文字列で書く。パースは`humantime`クレートに委ねており、`ms`・`s`（`sec`等）・`m`（`min`等）・`h`（`hour`等）に加え`ns`・`us`・`d`・`w`・`y`等も使え、空白区切りの複合表記（`"1h 30m"`）も書ける。単位は大文字小文字を区別し、小文字`m`は分だが大文字`M`は月（30.44日）を表す別単位になる点に注意する。詳細は[humantimeのドキュメント](https://docs.rs/humantime/latest/humantime/fn.parse_duration.html)を参照。

| フィールド | 必須 | 型 | 内容 |
| --- | --- | --- | --- |
| `initial_backoff` | - | 時間文字列 | 再起動間隔の指数バックオフの初期間隔 |
| `max_backoff` | - | 時間文字列 | 再起動間隔の上限 |
| `backoff_multiplier` | - | 整数 | バックオフの増加倍率。1以上 |
| `stable_uptime` | - | 時間文字列 | この時間以上稼働していたら安定稼働とみなし、バックオフを初期間隔にリセットする |
| `failure_uptime_threshold` | - | 時間文字列 | この時間未満での終了を失敗1回として数える |
| `max_consecutive_failures` | - | 整数 | 自動再起動を止めて失敗状態に遷移するまでの連続失敗回数。1以上 |

`initial_backoff`は`max_backoff`を超えてはならない。違反した場合や、時間文字列の形式が不正な場合、整数の下限を満たさない場合は設定読み込み時のエラーになる。

## spawn方式

[設計原則](./principles.md)のとおり、シェルを暗黙に挟まず`command`のargvをそのまま子プロセスとして実行する。

- `command[0]`が相対名の場合、appプロセスのPATH（LaunchServices経由の最小環境）で検索する（`posix_spawnp`相当）。
- appプロセスの環境が、ログインシェルの設定（HomebrewのPATH等）を含む保証は無い。ログイン環境が必要なサービスは`login_shell = true`を宣言する。
- 各サービスは専用のプロセスグループでspawnする。停止と孤児掃除の単位になる（後述）。

`login_shell = true`のサービスは、ログインシェルの初期化を経た環境で起動される。
挙動の契約は以下のとおり。

- argvはこの場合もシェル評価されず、記述したとおりに保たれる。
- `command[0]`の解決はログインシェルのPATHに従う。
- 環境はサービスの（再）起動のたびに再構築される。
- `env`で明示した環境変数は、ログインシェルの初期化結果より優先して適用される。この優先はshepherdrがspawn時に保証し、ログインシェルの初期化内容には依存しない。
- 作業ディレクトリは`cwd`の値（未指定ならappプロセスからの継承値）が最終状態になる。初期化スクリプトが`cd`を行っても影響しない。

## 監視と再起動

- 子プロセスが終了したら、終了理由を問わず再起動する。
- 連続クラッシュ対策として再起動間隔に指数バックオフを適用し、一定時間の安定稼働でリセットする（初期間隔・上限・リセット閾値）。
- 起動から一定時間未満での終了を、終了コード・シグナルによらず失敗1回とカウントする。
- 失敗が規定回数連続したら自動再起動を止め、そのサービスを失敗状態にする。トレイからの手動起動で失敗カウンタをリセットして再試行できる。
- 上記の具体値（初期間隔・上限・リセット閾値・失敗判定の経過時間・連続失敗の規定回数・バックオフの増加倍率）は、`config.toml`のトップレベル`[restart]`セクションで上書きできる。省略時は実装の既定値に従う（フィールド一覧は「設定ファイル」参照）。
- トレイからの停止はそのサービスのdesired stateを「停止」に変え、再起動の対象から外す。
- desired stateは永続化しない。app再起動後は常に`config.toml`の`enabled`に従う（[設計原則](./principles.md)）。

## 停止シーケンス

トレイからの停止・再起動、リロードに伴う停止、app終了時の停止は、いずれも同じシーケンスで行う。

1. サービスのプロセスグループへ`SIGTERM`を送る。
2. 猶予期間（実装の既定値）内に終了しなければ、プロセスグループへ`SIGKILL`を送る。

- app終了時は全サービスへこのシーケンスを並行して適用し、完了を待ってから終了する。
- 子孫プロセスが自らプロセスグループを作った場合はこの単位から外れる。子孫の完全な後始末はサービス自身の終了処理に依る。

## 孤児プロセスの掃除

app自体の異常終了では子プロセスは終了せず、孤児として残る。
次回起動時の重複起動を防ぐため、spawnしたサービスについてプロセスグループID・プロセス開始時刻・実行バイナリパス・OSのブート時刻をstateファイル（`~/Library/Application Support/shepherdr/`配下）に記録する。
app起動時の掃除は以下のとおり行う。

- OSのブート時刻が記録時と異なる場合（再起動を跨いだ場合）、孤児プロセスは生存し得ないため、stateを破棄して掃除をスキップする。
- 同一ブート内では、記録した開始時刻・実行パスをOS上の現在のプロセス情報と照合し、一致した場合のみ停止シーケンスで終了させる。プロセスグループIDが無関係なプロセスに再割り当てされていた場合の誤killを防ぐ。
- stateファイルが存在しない・読めない場合は、掃除をせず通常の起動を行う（初回起動はこの経路になる）。
- 停止シーケンスと同様に、自らプロセスグループを作った子孫は掃除の対象から漏れる。

掃除の完了後に、通常の起動フロー（`enabled`に従った起動）へ入る。
残存プロセスのadopt（監視の再開）はしない。親の終了でログ捕捉のパイプが失われており、再確立できないためである。

## 設定の再読み込み

設定の反映はトレイメニューからの手動リロードで行い、差分を適用する。

- 追加されたサービスは起動する。
- 削除されたサービスは停止する。
- 定義が変わったサービスは再起動する。
- `enabled`が変わったサービスは起動または停止する。

## ログ

- 各サービスのstdout/stderrをappが捕捉し、`~/Library/Logs/shepherdr/<name>.log`へ書く。
- サイズ上限と世代数によるローテーションをapp側で持つ。上限は`config.toml`の`[log]`セクションで上書きでき、省略時は実装の既定値を使う。
- ログウィンドウ（[UI](./ui.md)）はこのファイルを読んで表示する。
